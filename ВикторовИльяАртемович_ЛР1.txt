После добавить внутрь данного файла название файла и его разрешение, при индексировании он не будет найден

11) Сравнение

12) Удаление

13) Перемещение

14) Получение информации об истории коммитов
git log
git log --pretty=oneline (отображает наиболее полезные опции для изменения формата)

15) Отмена сделанных изменений и работа с метками
git checkout (хэш/тэг)
git tag (тэг)

16) Создание ветки, перемещение по ним, объединение и удаление
git branch (название) - создание новой ветки
git checkout -b (название) - создание новой ветки и переход на нее
git merge (название) - используется для слияния одной или нескольких веток в текущую, затем она устанавливает указатель текущей ветки на результирующий коммит
git mergetool - вызывает внешнюю программу слияний, в случае если у вас возникли проблемы слияния
git branch -d (название) - удаляет ветку

17) Механизм прятанья
git stash - прячет текущую ветку
git status - показывает состояния файлов в рабочей директории и индексе: какие файлы изменены, но не добавлены в индекс, какие ожидают коммита в индексе
git stash apply stash@{(номер скрытой ветки)} - возвращает ветку, для работы с ней нужно проиндексировать


18) Публикация на Github.com
git remote add origin git@github.com:ex3cute22/lab1.git
git branch -M master
git push -u origin master
